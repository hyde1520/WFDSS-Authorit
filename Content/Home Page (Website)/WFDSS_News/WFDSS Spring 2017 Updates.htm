WFDSS Spring 2017 Updates

The WFDSS Spring 2017 Update will be deployed on Training 6/7/2017 and to Production on 6/9/2017

New Updates:

  • Incident Groups:

  • From the Incident Groups List, you can now:

    • Click View Incident to review incident group information in Excel format by clicking the Excel icon to the left of the Incident Name column.
    • Click View Incident and click the hyperlink below the Current Decision columnto review decision status. If a published decision exists, the date is the hyperlink. If the first decision is pending, the hyperlink displays as "Pending." The column is blank for incidents without decisions.
  • Refer to Editing an Incident Group for further information.
  • Expanded Fire Behavior Specialist Role:
    • Fire Behavior Specialists will be able to set conditioning days to zero. This makes initial fuel moistures the same for every cell on the landscape, regardless of slope, aspect, elevation, shading and previous weather conditions. It should, generally, only be used for model testing or homogeneous, unshaded fuels on flat terrain).
    • Fire Behavior Specialists can view, copy and edit any analysis in the system without being added to the incident privileges.
    • Fire Behavior Specialists will have more downloads available on completed runs (All Models: Analysis Input. FSPRO: Daily Acres, Average Time Grid, Arrival Day Distribution, Suppression Effect, Ensemble Perimeters.)
  • See Fire Behavior Specialist for more information.

NOTE: As a result of these changes, all Fire Behavior Specialists are, ostensibly, Super Analysts. Be aware that some database and help content modifications need to be completed before the Super Analyst role is fully retired. This should occur prior to fire season 2018. Once modifications are complete, all users with the Super Analyst role will become Fire Behavior Specialists, but their capabilities remain the same.

  • New Report Templates:
    • A Risk Summary, Leader’s Intent Letter, Fire Behavior and Other report templates are now available.
  • Pages for optional content (Modeling and Benefits), when not populated with text by user, will no longer produced blank pages in the final decision document.
  • The fuel model alpha-numeric codes are now displayed next to the fuel model in the landscape editor drop-down list.
