Data Management

Find the Data Management perspective by clicking the Data Management tab. This tab is active for all users to view data associated with a unit, but users assigned the Data Manager role can choose a Planning Process for the unit and edit data. The top half of the screen displays the Unit Selection at all times. The bottom half of the screen is dictated by the tab selection, described below. 

All users can select different radio buttons for Geographic Area and Agency to populate the Unit list; however, the Unit list is dictated by your Contact Information under the My Home tab. For example, if your Contact Information under the My Home tab is the Eastern Geographic Area and the Agency is USDA Forest Service, no matter what radio buttons you select under the Data Management tab, the Units listed will always be Eastern Geographic Area/Forest Service units. Edit your Contact Information as needed to access unit data as required. Keep in mind that WFDSS keeps a log of users' actions as data are edited, so be nice and don't edit data if you don't have permission from the unit.

The Data Management tab has five sub-tabs.

  • FMU/SO Codes
  • Objectives
  • Unit Shapes
  • Unit Map
  • Notes

When you click a sub-tab, the content on the bottom half of the page changes. Data Managers can edit and/or view:

  • Fire Management Unit (FMU)/Strategic Objective (SO) Codes (FMU/SO codes sub-tab),
  • Strategic Objective and Management Requirement text from overarching Land, Resource, and Fire Management plans (Objectives sub-tab),
  • Unit Shapes to include Strategic Objective shapes, Management Requirement shapes, and Other Unit Shapes that contain information about local values data (Unit Shapes sub-tab),
  • Unit Shapes visually on the Unit Map (Unit Map sub-tab), and
  • Unit Fire Planning Map Layers (available for viewing on all map displays in WFDSS) (Unit Map sub-tab).

Data Managers can pre-load information for the first three items above, so the unit has information readily available during an incident. The Unit Map allows a quick check of recently uploaded unit shapes to ensure that each displays correctly . Before starting the upload process, you need the following information:

  • FMU or SO codes for your units and agency (FMU or SO codes must match the FMU or SO codes in the spatial data layer; FMU codes can be used still be used if there is not a spatial FMU layer)
  • General Management Plans
  • Land Management Plans
  • Resource Management Plans
  • Fire Management Plans
  • Local shapefiles frequently used in wildfire decisions

It is important to include this information so there is access during any decision-making process, and information is included correctly in the system of record for the incident.

Data Managers can click the Show History link (just below the Strategic Objectives/Management Requirements section) to review change history for a unit and if needed, filter information by entry date range, types, and/or actions. Results display Date, User, FMU, Action and Comments.

You can review the Data Layer reference information for all the available data layers. This information is updated frequently.
