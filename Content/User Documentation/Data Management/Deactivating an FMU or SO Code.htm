Deactivating an FMU or SO Code

Deactivating an FMU Code also deactivates any strategic objectives and management requirements associated with it. The information is no longer available to the decision process.

A Data Manger can deactivate a FMU code because FMUs are not required to have spatial data associated with them. The Strategic Objective (SO) Codes can not be deactivated since they are used in the spatial planning process. The SO codes can be modified by submitting a new SO data layer through your agency GIS representative.
