Analysis Notes

Experienced analysts understand the importance of documenting their thought process and assumptions when working with fire behavior models. All WFDSS analyst assisted models provide the option of adding Notes to any input, landscape, or results page. There are many reasons to provide documentation in the form of analysis Notes. For example, analysts may want to:

  • Provide justifications forvarious modifications to the default input values.
  • Document conversations with local subject matter expertsregarding modelinputs.
  • Leave a trail so that others can follow the thought process while moving through an analysis.
  • Explain analyst and/or model assumptions.
  • Describe changes and edits to a landscape.
  • Explain efforts to calibrate a model to known fire activity.
  • Provide interpretation of a models outputs and express confidence, caveats and any concerns with them.
  • Create reference documentation to assist with incidents in the future orin the event of investigations or litigation.

Once created, notes appear at the top of each page that they were added to (user selects the page they will display on from a drop-down list when creating them). You can add, edit, delete, or import notes any time before an analysis is run or anytime after, regardless of the status of the analysis.

You can access notes from within individual runs or from the Analysis List for individual analyses. This is helpful for comparing runs and input logic. Analysis Notes can be viewed but not edited from the Analysis List.
