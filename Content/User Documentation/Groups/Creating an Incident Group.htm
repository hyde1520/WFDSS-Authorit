Creating an Incident Group

WFDSS users assigned the role of Author can create public or private Incident Groups using a two step process. Incident Groups are also created automatically for grouped incidents (complexes) in IRWIN.

You can create an Incident Group two ways:

  • By naming the Incident Group first, then add incidents to it that you select.
  • By drawing a polygon around incidents onthe Intelligencemap.

An Incident Group is limited to 80 incidents in both Production and Training. Each Incident Group can be private or public. Private Incident Groups are available only to you, but you can share ownership if appropriate. Public Incident Groups are available to all WFDSS users. You can change an Incident group from private to public and vice versa if you are a group owner. Only a group owner or a National Editor can delete a group.

Once created, you can:

  • Add or remove incidents from the Incident Group.
  • Transfer Incident Group ownership.
  • Share ownership with multiple owners.
  • Filter groups by group name, owner, state and/or group type (public or private.)
  • View incidents in the Incident Group.
  • View details, situational information, and analyses for the Incident Group.
  • Create map bookmarks for the Incident Group.

Note: You cannot run analyses, reports, or create decisions for a group of incidents. Analyses, reports, and decisions are created and processed for individual incidents using the WFDSS Incidents tab's Decisions, Reports, and Analysis sub-tabs.
