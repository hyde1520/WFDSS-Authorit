You must document a decision in the following circumstances:

  • The current course of action is not valid and requires a new course of action.
  • The situation requires a change in the level of oversight, significant changes to resource requirements, or increased costs require escalation to the next level of authority.
  • The incident has escaped initial attack and is escalating into extended attack or long-duration event.

Before publishing a decision, you need the following information:

  • Enter the Unique Fire Identifier for the incident when you create an incident or edit an incident. TheUnique Fire Identifier mustcontain a Unit ID that ends with the local numberfor the incident. After a decision is published, the Unique Fire Identifier is locked.
  • Enter the Incident Size (Acres) that is greater than zero when you create an incident or edit an incident.
  • Enter at least one Decision Approver when you create an incident or edit an incident.
  • Designate at least one approver for the decision when you create an incident or edit an incident.
  • Designate a Planning Area shape file. If onedoes not already existyou must create a planning area.
  • Include the appropriate strategic objectives and management requirements.
  • Include incident objectives and requirements. If these requirements do not already exist, create incident objectives and requirements, or edit incident objectives and requirements.
  • Identify and include at least one action item for the course of action.
  • Estimate final cost for the incident.
  • A relative risk calculation for the incident.
  • An Organization Assessment for the incident.
  • Add content to the Rationale section of the decision using the Decision Editor.
