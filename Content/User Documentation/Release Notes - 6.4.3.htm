Release Notes - 6.4.3

10/23/19 - Bug Fixes for Release 6.4.3

  • The name on the Assign MAP and Create MAP popup windows are displaying accurately.
  • Landscape masks are now visible when viewing analysis results or view landscape maps.
  • Alaska strategic objectives are now listed only once when a planning area includes two or more units.
  • You can now move incident objectives and/or requirements up or down, but are limited by the default number of rows displaying on a page.
  • Removed the Planning Area Download button from the Incident Group pages. The button was mistakenly added in a previous release and did not function.
  • The zip file now populates when downloading decisions from the Incident Groups>View Incidents page.
  • Users can now draw and save a shape after uploading a shape.
  • Whenthe name of aRAWS displayed a apostrophe, the decision editor displayed an error message. All apostrophes are removed from all RAWS in WFDSS.
  • Adjusted the hover text for the Download Decision button under Incident Groups.
  • Correct roles show for accounts disabled prior to August 28, 2019.
  • Fixed a bug in which the fuel model histogram in a landscape of a fire behavior analysis cached preventing it from updating when the landscape was edited and a new landscape generated.
  • An error message displays when there are two MAP’s for two separate incidents with the same name and MAP number. This happens switching between two incidents during the same session.
  • Clarified the description on where to find downloaded KMZ files. On the submit request page it now states, "once the request completes, you access the results from your 'My Home > Sub Tab My Activities'".
  • Adjusted the Flame Act language for accuracy.
