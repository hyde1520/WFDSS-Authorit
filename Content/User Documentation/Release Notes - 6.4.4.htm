Release Notes - 6.4.4

3/4/2020 - Bug Fixes for Release 6.4.4

Account Management

  • Resolved an issue with passwords not being able to be reset with certain workflow.

Analysis

  • Corrected an issue which resulted in a 500 error when analyses are completed for an incident in which IRWIN, rather than a user, is the Owner.
  • Adjusted character length limits for input values in the Landscape Editor that were incorrectly limiting users.
  • Fixed a bug in the Analysis Filter that prevented the Completion Date Calendar from functioning correctly.
  • Corrected a bug with the FSPro Values at Risk outputs that incorrectly calculated the Expected Value (EV) acreage for Management Requirement shapes.
  • Modified the layout of the Fire Behavior Request screen so text wraps and the calendar is not offset.
  • Corrected the STFB Analysis output to display the arrival time acreage correctly instead of displaying the entire landscape acreage.
  • Fixed the Get Forecast button on the FSPro ERC Stream page to ensure users can generate NWS forecasts for FSPro runs for areas in which the primary forecast retrieval method is not working.
  • Fixed a bug in which the NWS forecast would yield an ERC value of -1 if a user did not click save on the FSPro Information page prior to generating a forecast.

Correspondence: Emails, Feedbacks, and Subscriptions

  • Corrected an issue causing multiple copies of a single Feedback to be generated.
  • Changed Feedback response texts and emails to Users and Support Team to previous format to include contact information of the original user (not their User ID) and added the Feedback ID to the Info Panel.
  • Fixed a bug in Feedback Notification Subscriptions that prevented users from subscribing by email.
  • Updated format to remove the extra line between Analysis Status and Brower in Subscription texts/emails of Feedbacks.

Data Management

  • Updated the Jurisdictional Agency legend to match the agencies and colors displayed on maps.
  • Adjusted the Alaska Strategic Management Objectives to ensure Planning Areas drawn over multiple units only list the Objectives once in the decision and on the Objectives tab.
  • Updated so that the Unit dropdown and other menus work on Apple mobile devices.

Incidents

  • Fixed a bug with the Transfer Filter feature, ensuring filters transfer correctly.
  • Adjusted the Cost field in the Management Action Point (M.A.P) screens to prevent failure messages when a user enters too large of a value; correct error messages now display.
  • Corrected the notes for historical Organizational Needs Assessments (ONAs) to ensure users can access historical assessment notes instead of only current Assessment notes.
  • Required fields cannot contain null values; users will receive a warning message. This includes fields related to Management Action Points (Conditions/Actions), Course of Action (Description), Incident Objectives and Management Requirement (Description), Strategic Objectives and Management Requirements (Description).
  • Adjusted character limiter lengths for input values for Estimated Final Fire Size in Stratified Cost Index (SCI) and Estimated Final Cost to ensure consistency.
  • Adjusted the analysis and value requests so that only one request is submitted regardless of user ID to prevent simultaneous requests for the same analysis, planning area, point of origin, and perimeter.

Incident Groups

  • Ensured the Download Planning Area button remains out of the application until the functionality can be implemented correctly.
  • Corrected the sorting on Incident Groups Name Filter feature to ensure it sorts Incident Groups alphabetically.
  • Fixed a bug in Incident Groups to allow all owners of a group to modify any aspect of the group.
  • Fixed a bug in which the Points of Origin for Incidents displayed the previously viewed group and not the currently selected group.
  • Fixed a bug that prevented the correct users from displaying when using the Transfer Ownership button. Transfer can occur to all National Editors, GAEs for the area the incidents are in, and users who have been granted ownership to all of the incidents in the group.
  • Fixed a bug to ensure the KMZ Download feature from the Incident Group Map includes Incidents that are associated with the current Incident Group and does not include Incidents that last appeared in the Incident Group List.
  • Corrected typo "Grorup" to "Group" on Incident Group left-hand fly-out menu.
  • Added the Incident name to the file name to prevent Group Decision download errors caused by duplicate file names that occur when different decisions in the same Incident Group are published simultaneously.

Support Functions (Administrators)

  • Modified to display which service broker is processing a transaction (Analysis, Value, Landscape, etc.) instead of only updating when completed to provide better trouble shooting.
  • Adjusted the Reset GIS button to prevent error messages when clicked multiple times.
  • Updated format for the Weather Related utilities. ERC stations and Wind stations emails now include a range of stations processed, and the emails now consistent with the ones for the ERC-G station utility.
  • Added the ability to update data for a single weather station including Winds, ERCs, and the ERC Graph.
  • Added the ability to generate an LCP Critique for an analysis when the landscape file does not generate correctly.
  • Added the ability to regenerate the PDF/HTML for a published decision when it doesn’t publish correctly (with tracking of user who performed the action and a comment to explain why the files needed to be regenerated).
  • Added the ability to Return a 'stuck' Analysis to Assigned (with tracking).
  • Added the ability to Return a 'stuck' Decision to available (with tracking).

Weather

  • Updated the Hourly Weather Forecast link to current NWS site (NWS decommissioned previous link).

WFDSS Lite

  • In the Development and QA environments, the login URL for the decommissioned WFDSS Lite application was removed from Periodic Assessment emails.

IRWIN Enhancements and Fixes

Defects Fixed for the WFDSS/IRWIN Data Exchange

  • When the Push to IRWIN button is clicked, any updated information in WFDSS will be updated in IRWIN correctly.
  • The Estimated Final Cost in IRWIN will be populated with the most recent published cost in WFDSS (the cost associated with the most recent published decision or periodic assessment validating that decision).
  • Incidents received from IRWIN as Wildfires will continue exchanging data with IRWIN if their type is changed to Other. Wildfires created in WFDSS that change to type Other or Prescribed will continue to exchange data with IRWIN.
  • Users will no longer be able to modify or delete an Incident Group created from an IRWIN Complex. The Incident Group will maintain the list of Incidents within the IRWIN Complex when the complex is updated.
  • Incidents that have been marked Certified in IRWIN can now be modified in WFDSS, though only for fields in which WFDSS has authority to write to, based on IRWIN authoritative source hierarchy.

Enhancements Incident History List

  • The following actions will now be tracked: Jurisdiction a Point of Origin set, IRWIN is Valid updated, Incident Pushed to IRWIN, Incident Pulled from IRWIN, and IRWIN GUID Set.
  • The action, IRWIN isQuarantined updated, has been changed to record when WFDSS was notified by IRWIN that the incident was marked as Quarantined.
  • Hover text has been added to the Date Column in the Incident History List to display the timestamp including seconds of the action to aid in problem solving.

Enhancements Incident Information Page - IRWIN Information Section for National Editors

  • A checkbox has been added to allow a user to mark an Incident as Valid or Invalid in IRWIN.
  • Unselecting the 'Incident Quarantined in IRWIN' on an incident created in WFDSS will remove any potential conflict relationships.
  • A 'Compare WFDSS and IRWIN Attributes' link has been added that will display a new popup window that shows the WFDSS Incident Information alongside its corresponding information in IRWIN. This feature will improve troubleshooting.
